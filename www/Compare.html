Compare time series

 The 
'Compare'
 module supports time series comparison.

 The 
'Data to compare'
 menu allows to select data to compare.
 The menu list includes the input, detrended and model data generated by application.

 Use 
'Comparison method'
 menu to select the comparison method :

   
'Autocorrelation'
  method to compare autocorrelation function (ACF) parameters,

   
'Crosscorrelation'
 method to compare crosscorrelation function (CCF) parameters,

   
'Synchrony'
        method to estimate synchrony between time series (see 
'Synchrony' note
 at the page bottom).

 If the 'Autocorrelation' method is selected, the 
'Confidence bands'
 parameter is available. 
 The 'Confidence bands' parameter determines how 95% confidence bands will be calculated:

   
'Standard formula'
     - using standard formula for the standard error.

   
'Bartlett's formula'
   - using the Bartlettâ€™s formula for the standard error.

 Click 
'Compare'
 button to compare the selected data. 
 The 
'Comparing data ...'
 message appears and blinks during the process.                

 The result will be presented graphically in the 
'Comparison plot'
 window, which will contain plots and tables.
 Under the plot window the plot options will appear.

 'Autocorrelation' method results are: 
  
'Input data to compare'
 and 
'Autocorrelation function'
 plots,
  
'Select traces to plot:'
       option - determines the traces to plot,
  
'Autocorrelation to plot:'
     option - indicates the trace for the 'Autocorrelation fuction' plot,
  
'Significant autocorrelations'
 table  - contains the numbers of all autocorrelations (except lag 0) that are 
        non-zero at the 95% confidence interval. The table values depend on the paramater 'Confidence bands'.
   For more info, please see 
'Autocorrelation' note
 at the page bottom. 
        
 'Crosscorrelation' method results are: 
  
'Input data to compare'
 and 
'Crosscorrelation function'
 plots,
  
'Select traces to plot:'
           option - determines the traces to plot,
  
'Crosscorrelation to plot:'
        option - indicates the traces for the 'Crosscorrelation fuction' plot,
  
'Crosscorrelation martix to plot:'
 option - selects which table will be displayed:
        
'Crosscorr. at lag0'
           table  - cross-correlations at lag zero,
        
'Crosscorr. maximum'
           table  - maximal cross-correlations,
        
'Lag for maximum crosscorr.'
    table  - lags at which the maximum of the CCFs occur.
   For more info, please see 
'Crosscorrelation' note
 at the page bottom.

 'Synchrony' method results are:
  
'Input data to compare'
 plot,
  
'Synchrony'
 table contains the values from -1 to 1, where -1 is a perfect negative synchrony,
        0 no synchrony and 1 a perfect positive synchrony.
   For more info, please see 
'Synchrony' note
 at the page bottom.
   
 The help window appears when 
'Help'
 button is pressed. Click the 
'Close'
 button to close the window.

 On an error, the error window could appear (see 
'Error window' warning
 at the bottom of the page). 

 
Notes:

  
'Autocorrelation': Confidence bands calculation.

    There are two formulas for the ACF confidence bands calculation. For details, see ACF_confidence_bands.pdf.
    The values in the 'Significant autocorrelations' table depend on the formula selected. Please note that 
    the number of significant autocorrelations excludes the autocorrelation at lag 0, which is always 1.
  
'Crosscorrelation': CCF parameters.

    The crosscorrelation at lag zero, 'Crosscorr. at lag0', is actually the Pearson's correlation coefficient. 
    The possible spurious correlations should be kept in mind when interpreting the 'Crosscorr. at lag0' values.

    The maximum of the CCF, 'Crosscorr. maximum', indicates the point in time where the time series are best aligned.
    The lag at which the CCF reaches its maximum, 'Lag for maximum crosscorr.', when multiplied by the sampling interval,
    is the time delay between time series.

  
'Synchrony': Synchrony calculation.
 
    The synchrony between two time series is a value from -1 to 1, where -1 is a perfect negative synchrony,
    0 no synchrony and 1 a perfect positive synchrony. The synchrony estimates how often the changes in the time series 
    under comparison have the same sign. For details, see Synchrony.pdf.
